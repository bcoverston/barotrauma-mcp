# LuaCs API Verification

> Generated by multi-agent verification (4 researchers + synthesis) against the
> LuaCsForBarotrauma `master` HEAD (the `latest` rolling build, ~2026-06-18) and
> the vanilla Barotrauma source. **Version-sensitive** — re-verify if the
> installed build differs. Items marked "must check in-game" are unresolved from
> docs alone; the mod's load-time **write probe** settles the most important one.

## TL;DR — action items

1. **DONE (critical):** the v0 mod wrote `state.json`/`ack.json` to a game-root
   path the LuaCs sandbox blocks, so the bridge was silently dead. Repointed IO
   to `LocalMods/AgentBridgeIO/` (a writable root) and added a load-time write
   probe that prints OK/FAIL. → `agent_bridge.lua`, `agent/src/bridge.js`.
2. **DONE (recommended):** replaced the FPS-dependent frame-count throttle with a
   wall-clock `Timer.GetTime()` interval (~0.5s independent of frame rate).
3. **Confirmed as-is:** `Speak` signature, all read fields, the `tostring()`
   guards on `JobIdentifier`/`DisplayName`, `Game.RoundStarted`, `Timer.GetTime`.
4. **M0 still needs in-game confirmation of:** the write probe result, the
   `Speak` line actually appearing (it returns void; `ok=true` ≠ spoken), and
   `File.CreateDirectory` on the installed Mac patch.

## 1. Feasibility

**Verdict: Yes — the macOS *client* patch runs LuaCs in singleplayer, so M0 can
be tested on this Mac.** The release ships `luacsforbarotrauma_patch_mac_client.zip`
as a first-class CI target (no "experimental" qualifier), and the client patch is
precisely the one that "allow[s] you to use the mod in singleplayer." Install is a
drop-in DLL patch, not a code build — extract the patch into
`Barotrauma.app/Contents/MacOS/` (right-click the app → *Show Package Contents* →
`Contents/MacOS`), then verify with the F3 console command `cl_reloadluacs`. The
`think` hook fires per-frame client-side in singleplayer, so the tick loop runs
without a dedicated server.

Two Mac-specific cautions, neither blocking: (a) the game's working directory is
`Barotrauma.app/Contents/MacOS/`, so the external agent/MCP must point at paths
relative to there; (b) macOS Gatekeeper/quarantine (`xattr`) behavior on a patched
binary is undocumented — a plausible first-launch friction point.

## 3. Required changes to agent_bridge.lua

- **CRITICAL — write path (applied).** `DIR = "AgentBridge"` was game-root-relative.
  Writes are permitted only under the save folder, `LocalMods/`, `WorkshopMods/`,
  and (client) the mod-download folder; the game root has no write clause and the
  LuaCs package dir is explicitly blocked. → Repointed `DIR` to
  `LocalMods/AgentBridgeIO`, added a load-time write probe that prints OK/FAIL.
  - ⚠️ **Source conflict, called out:** Researcher 1 inferred a "looser" game-dir
    sandbox (but could not fetch the source); Researcher 3 read `CanWriteToPath`
    directly (from `master-backup`, referenced unchanged by master) and found no
    game-root write clause. We weight the direct read and treat the game-root
    write as broken. Writing under `LocalMods/` is safe under *either* reading, so
    the fix holds regardless — the probe confirms empirically on first load.

- **RECOMMENDED — wall-clock throttle (applied).** `TICK_FRAMES = 30` gave an
  FPS-dependent cadence. Replaced with a `Timer.GetTime()` interval (tick when
  `now - lastTick >= 0.5`) for a stable ~0.5s tick.

- **CONFIRMED AS-IS — no change:** the `Speak` call; `tostring(ch.JobIdentifier)`
  and `tostring(ch.CurrentHull.DisplayName)`; the `CurrentHull`/`CharacterList`/
  `CreateDirectory` nil-guards; `OxygenAvailable` (unless intent is O2-deprivation
  health, then `ch.Oxygen`).

## 4. For M2 (order) / M3 (console)

**Order shape.** Build an `Order` from an `OrderPrefab`, then
`character:SetOrder(order, isNewOrder, speak=true, force=false)`. `SetOrder` has a
**hearing gate** — `if (!force && orderGiver != null && !CanHearCharacter(orderGiver)) return;`
— so pass `force=true` (or build with `orderGiver=null`) or distant bots silently
ignore the order.

- Prefab lookup: `OrderPrefab.Prefabs["operatereactor"]` (string indexer).
- Order ctors: `Order(prefab, targetEntity, targetItemComponent, orderGiver=null, isAutonomous=false)`
  and an option-bearing `Order(prefab, option, targetEntity, targetItem, orderGiver=null, ...)`.
- Confirmed identifiers in this build: `operatereactor, fixleaks, steer,
  extinguishfires, fightintruders, follow, wait, operateweapons, cleanupitems,
  repairelectrical, repairmechanical, repairsystems, reportbrokendevices`.
  Dismissal = `dismissed`. **It is `fixleaks` (plural)** — `fixleak` does not exist.
- Inspect current order for the snapshot: `ch:GetCurrentOrderWithTopPriority()?.Identifier`
  (skips `dismissed`/priority<1). `CurrentOrders` can be null when `Info` is null
  (monsters) — guard it.

```lua
-- order a bot to operate the reactor
local prefab = OrderPrefab.Prefabs["operatereactor"]
local order  = Order(prefab, prefab.Options[0], reactorItem, reactorComponent)
-- optional: order = order:WithManualPriority(N):WithOrderGiver(Character.Controlled)
bot:SetOrder(order, true, true, true)   -- isNewOrder, speak, force

-- additive snapshot field (no schemaVersion bump):
order = safe(function()
  local o = ch:GetCurrentOrderWithTopPriority()
  return o ~= nil and tostring(o.Identifier) or "none"
end, "none")
```
For hull/leak orders, construct target-less:
`Order(OrderPrefab.Prefabs["fixleaks"], hull, nil)` and let the bot AI find leaks.

**Console shape.** Call `Game.ExecuteCommand(string)` — LuaCs's wrapper over
`DebugConsole.ExecuteCommand` (`LuaGame.cs:402`); the raw `DebugConsole` is **not** a
Lua global (it throws "index a nil value"), so go through `Game`. Returns `void`, so the
ack only asserts "dispatched without a Lua error," not "succeeded." Cheat-gated commands
(`spawnitem`, `fire`, …) need a prior `Game.ExecuteCommand("enablecheats")`. The mod
gates the verb behind the sentinel file `LocalMods/AgentBridgeIO/console.enabled`
(default off), not a code flag.

## 5. Open uncertainties (check in-game)

1. **The write-path sandbox (highest priority).** R1/R3 conflict; neither read the
   live file on master HEAD. The load-time probe settles it; until it prints OK,
   treat the write as suspect. Manual test: `print(pcall(function() File.Write("AgentBridge/x.json","1") end))`
   from the game root vs under `LocalMods/<pkg>/`.
2. **`File.CreateDirectory`** presence/behavior on the *installed* Mac patch.
3. **`OxygenAvailable` vs `Oxygen`** runtime ranges (semantic, not a correctness bug).
4. **MoonSharp marshalling** of the `Speak` 4th arg (`""` → `Identifier`), and for M2
   the `string→Identifier` conversions and `OrderPrefab.Prefabs[...]` string indexer
   across the LuaCs reflection boundary.
5. **Whether `OrderPrefab`/`Order` helpers** (`GetMatchingItems`, `WithItemComponent`)
   and `GetComponent` are exposed to Lua, or reflection-filtered (fall back to
   iterating `Submarine.MainSub` items).
6. **`think` fires every client frame in SP** on this build — established LuaCs
   behavior, not re-verified against a running game.
7. **macOS Gatekeeper/quarantine** on the patched binary.
8. **Version pinning** — all facts from `master` HEAD, not a tagged release.
